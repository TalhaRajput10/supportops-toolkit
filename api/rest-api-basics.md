# REST API Basics

## What is a REST API?

A REST (Representational State Transfer) API is a set of rules that allows different software applications to communicate over the internet using HTTP requests.

Example:
- A mobile banking app requests your account balance from the bank's server.
- A food delivery app requests nearby restaurants from its backend.
- An e-commerce website retrieves product information from a database.

These interactions are powered by APIs.

---

# HTTP Methods

## GET

Retrieves data from a server.

Example:

GET /users

Use Case:
Retrieve customer information.

---

## POST

Creates new data.

Example:

POST /users

Use Case:
Create a new customer account.

---

## PUT

Updates an existing resource.

Example:

PUT /users/123

Use Case:
Update a customer's profile.

---

## DELETE

Deletes a resource.

Example:

DELETE /users/123

Use Case:
Remove a user account.

# Why REST APIs Matter

Technical Support Engineers regularly troubleshoot API-related issues.

Understanding REST APIs helps with:

- Investigating customer issues
- Reading API documentation
- Testing endpoints
- Identifying authentication problems
- Working with developers

# Summary

REST APIs are one of the most important technologies used in modern software. A strong understanding of REST concepts is essential for Product Support and Technical Support Engineers.
